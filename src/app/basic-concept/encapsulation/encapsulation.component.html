Encapsulation in Angular

Encapsulation in angular to the restriction of css code. It providing the ability to keep the particular part css only applies on that part and not affect to the other component or any other part of code. Usually there are three types of encapsulation in angular which is:

  • Emulated: No Shadow DOM. Style encapsulation. (default) It usually means that this component css will not applies on their parent or child or used in the this component only.
  • Native: Shadow DOM Style encapsulation. This option is used on that time when we want to apply the this component css to its child component but not apply on its parent component. Native option created a shadow dom for its child where all the parent css is defined.
  • None: No Shadow DOM. No Style encapsulation. It means simply that this component css will be applied on its parent and child component both.
