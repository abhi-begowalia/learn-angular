Data Binding

Data Binding means to bind the view (HTML Content) with Controller's (Component) field. That is whenever we display dynamic data on a view (HTML) for Component, data binding is used.

Data Binding basically means intracting with data. So, we can say that the interaction between templates and business logic is called data binding. There are basically two types of Data binding in Angular

  • One way Binding
    • Component to View
      • Interpolation Binding
      • Property Binding
      • Style Binding
      • Class Binding
      • Attribute Binding
    • View to Component
      • Event Binding
  • Two way Binding

Interpolation Binding

Interpolation Binding is a technique that allows the user to bind a value to a UI element. Interpolation Binds the data in One way Which is Component to View. This means that when the value of the field bound is changed, It update the page as well.

Property Binding

It is used to bind value of Component/model properties to the HTML element. Depending on the values, it will change the existing behavior of the HTML element. Syntax [property]='expression' In the property binding, there is a source and target. for example, we can define it as [innerHTML]='firstname'. Here, innerHTML is a target that is a property of span tag and source is a component property i.e. firstname. It is also a one way data binding.

We must always use property binding because Interpolation get fails when we work with boolean value.
